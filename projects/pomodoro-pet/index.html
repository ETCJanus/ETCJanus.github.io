Pomodoro Pet

Working Prototype
Flutter / Dart Hive DB Provider Android / iOS Pixel Art

A mobile app that combines a Pomodoro focus timer with Tamagotchi-style pet care. Instead of relying on willpower alone, your virtual pets depend on you to study — skip a session and they go hungry. Complete one and you earn gold to feed, customize, and grow them.

I built this because I used apps like Forest but always wanted something with more emotional stakes. A tree growing is nice, but a pet that needs you? That felt like real motivation. It was also a way to teach myself Flutter and mobile development from scratch.

> THE_CONCEPT_

Apps like Forest grow a tree while you focus.
That's nice. But can we add more emotional value to this?

THE IDEA: What if your pet's happiness depended on your focus?
Study = pet gets fed. Skip = pet goes hungry.

You're not studying for yourself anymore.
You're studying for them. 🐾

What It Does

Features

⏱️

Focus Timer

A customizable Pomodoro timer with work and break intervals. Completing a session earns gold and XP for your pets.

🚫

App Blocking

During a focus session, the app monitors if you open blacklisted apps (socials, games, etc.). If you do, your pet loses happiness. Uses Android's UsageStatsManager API and required working through the permissions system.

🐱

Virtual Pet System

Your pets have hunger and happiness stats that decay over time. Buy and own multiple pets, give them custom names, and pick their appearance.

🏪

Economy & Shop

Gold earned from studying buys food, items, and new pets from the in-app shop. The economy is balanced so you need consistent sessions to keep your pets well-fed.

🎨

Pet Customization

Choose your pet's look and name from a personalization screen. Leveling up unlocks new skins — a long-term motivator.

🌙

Dynamic Day/Night

The home scene changes based on the real time of day. Afternoon shows a sunny sky; evening shifts to a moonlit night. All hand-drawn pixel art.

Under the Hood

Technical Architecture

Building a mobile app from scratch revealed how much goes on behind the scenes, even for something that looks simple on the surface.

State Management

The app uses Provider to manage interconnected state across the timer, pet stats (hunger, happiness), and the economy (gold, inventory). These systems need to stay in sync, especially when the timer is running in the background while the app is minimized.

Local Persistence

All data is stored locally using Hive, a lightweight NoSQL database for Flutter. Pet data, user progress, and inventory persist between sessions without needing an internet connection or backend.

App Usage Monitoring

The blocking feature required working with Android's UsageStatsManager API and navigating the permissions system. Getting the right permissions to monitor foreground apps was one of the trickiest parts of the project.

Build Pipeline

Setting up the full build pipeline — from writing Dart code to producing a working APK/IPA — was a learning experience in itself. Understanding signing, manifests, and platform-specific configuration added a new level to my development skills.

Reflection

What I Learned

  • Mobile architecture is its own world. Coming from web development, I had to learn about platform channels, lifecycle management, permissions models, and background services — things that don't exist on the web.
  • Permissions are harder than the feature itself. Getting app usage monitoring to work was less about the code and more about understanding Android's layered permission system and guiding the user through granting access.
  • Pixel art takes real effort. What looks "simple" on screen took hours of careful, pixel-by-pixel drawing. I have a much bigger appreciation for game artists now, even at a small scale.
  • The build pipeline matters. Going from source code to a working app on a physical phone involves signing, platform configs, and debugging environment-specific issues — a whole skill set on its own.
  • Emotional design works. Using empathy as a motivator (caring for a pet) instead of abstract goals (tree growth) felt more engaging, even for me as the developer. It validated the core idea.
