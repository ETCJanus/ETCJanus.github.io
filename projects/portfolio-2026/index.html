The "Living Lab" Concept

I didn't want to just list my skillsâ€”I wanted to demonstrate them. This website serves as a living laboratory where I experiment with interaction design, creative coding, and performance optimization.

Instead of relying on heavy frameworks like React or Three.js for simple tasks, I challenged myself to build the core visual engines using Vanilla JavaScript and the HTML5 Canvas API. This ensures maximum performance (60fps) while keeping the bundle size microscopic.

The Interaction Engines

The site is powered by three distinct "Canvas Engines" that I wrote from scratch. Each one serves a thematic purpose corresponding to the page content.

1. The Constellation Engine

Location: Home Page

An interactive particle system where nodes connect dynamically based on proximity. It represents "Networking" and "Complexity," reacting to mouse movement with a repulsion force.

2. The Cyber HUD

Location: About Page

A facial analysis overlay that tracks the DOM element of my profile picture. It draws scanning lines and hex-code data streams, symbolizing my role as a bridge between Human and Machine.

3. The Flow Stream

Location: About Background

A Matrix-inspired vertical stream of code symbols ({ }, </>, 01) that float upwards, creating a sense of depth and continuous motion without distracting from the text.

Technical Highlight: The "Strict Necessity" Logic

One of the biggest challenges was managing performance. Running three full-screen canvases is expensive. I implemented a strict Observer Pattern logic:

// Only init the effect if the canvas actually exists on the current page const heroCanvas = document.getElementById('hero-canvas'); if (heroCanvas) initHeroEffect(heroCanvas); // Inside the animation loop: // Smart clean-up to prevent memory leaks if (!activeParticles) cancelAnimationFrame(animationId);

This ensures that scripts for the "About" page don't burn CPU cycles when the user is on the "Projects" page.
