Overview

As I was finishing my Master's in Interaction Technology, I needed a professional portfolio to showcase my work for job applications. Rather than using a template or website builder, I decided to build it from scratch as a way to demonstrate my actual skills in web development, interaction design, and visual creativity.

The challenge was to create something that felt unique and interactive without relying on heavy frameworks. I wanted potential employers to see not just what I've worked on, but how I think about technical implementation and user experience.

The Concept

I wanted the portfolio to feel like a living demonstration rather than just a static resume. The idea was to use the website itself as proof of my technical abilities. If someone lands on the home page and sees interactive particle systems running smoothly, that tells them more about my JavaScript skills than any bullet point could.

I decided to build everything using vanilla JavaScript and the HTML5 Canvas API instead of reaching for Three.js or React. Not because those tools are bad, but because I wanted to show I understand the fundamentals and can build performant interactive experiences from first principles. It also keeps the site fast and lightweight, which matters for both user experience and that crucial first impression when someone visits.

What I Built

The site has three main pages (Home, Projects, About), each with its own interactive canvas effect that runs in the background. Writing these from scratch taught me a lot about performance optimization and managing animation loops.

Constellation Effect (Home)

An interactive particle system where floating nodes connect when they're close enough to each other. It responds to your mouse position by pushing particles away, creating a nice interactive feel. I built this entirely in canvas using basic physics for particle movement and distance calculations for the connecting lines.

Cyber HUD Overlay (About)

A scanning overlay that tracks my profile photo and draws animated lines and data streams around it. This one was trickier because it had to sync the canvas coordinates with the actual DOM element position, especially when the page resizes or scrolls.

Code Stream (Projects)

A matrix style effect with code symbols flowing upward in the background. It adds motion and visual interest without being distracting. Each column moves at a slightly different speed to create depth.

Technical Challenges

Getting everything to work smoothly across different pages and screen sizes turned out to be more complex than I expected. Here are some of the main challenges I worked through:

Performance Management

Running full screen canvas animations at 60fps is expensive. I had to make sure that only the canvas effect for the current page actually runs. If you're on the About page, the code for the Home page constellation shouldn't be burning CPU cycles in the background. I set up conditional initialization so each effect only starts if its canvas element exists on the page.

Responsive Canvas

Canvas elements don't automatically respond to window resizing the way normal HTML does. I had to write resize handlers that not only update the canvas dimensions but also recalculate particle counts and positions. Getting this smooth without causing visual glitches took several iterations.

DOM and Canvas Coordination

For the cyber HUD effect on the About page, the canvas needs to track where my profile image is positioned on the screen. This meant getting the bounding rectangle of the DOM element and translating those coordinates to canvas space, all while handling scrolling and viewport changes. This was definitely the most complex effect to get right.

Design System

I wanted a cohesive visual identity that felt modern and tech focused without being too overwhelming. The design uses dark backgrounds with glassmorphism effects (those frosted glass style cards you see throughout the site) and accent colors that combine purple and teal gradients.

All the styling is custom CSS. I created utility classes for consistent spacing, text styles, and card components so I wouldn't have to keep writing inline styles everywhere. The glassmorphism effect uses CSS backdrop filter, which required making sure browser compatibility was acceptable (it works in all modern browsers but not older ones).

What I Learned

Building this portfolio from scratch turned out to be one of the most valuable learning experiences of my master's program. It forced me to really understand how web technologies work at a fundamental level, not just how to use a framework.

Working with the canvas API taught me about animation loops, coordinate systems, and performance profiling. I learned when to optimize and when performance is already good enough. I also got much better at debugging visual issues, since canvas problems don't show up in the browser inspector the same way DOM issues do.

The project also reinforced the importance of planning your CSS architecture from the start. I initially had a lot of inline styles scattered everywhere, which made updates painful. Refactoring everything into a proper design system with utility classes took time but made the codebase so much more maintainable.

Most importantly, I realized that your portfolio itself can be evidence of your capabilities. When I send this site to potential employers, they can see I know how to build performant, interactive web experiences before they even read about my other projects.
